Two coordinated landings on top of `adb0f01`:

1. **Pre-commit hook for spec integrity**. New `.githooks/pre-commit` runs `npm run validate --workspace=@skill-map/spec` only when the staged change touches `spec/` (silent no-op otherwise) — catches the desync class where `spec/conformance/coverage.md` is edited without regenerating `spec/index.json`. New `prepare` script in root `package.json` runs `git config core.hooksPath .githooks`, so any clone gets the hook wired automatically on `npm install`. `CONTRIBUTING.md` and `context/scripts.md` document the hook + auto-wiring.

2. **`scripts/*.js` migrated into the workspaces that own each script**. The architectural rule (formalized in `context/scripts.md`): a `.js` in root `scripts/` is justified only if genuinely cross-workspace; otherwise it lives inside the workspace and is exposed via that workspace's npm scripts. Applied to: `spec/scripts/build-index.js` + `sync-pin.js` + `check-coverage.js` (renamed to drop the redundant prefix); `src/scripts/build-reference.js` + `dev-serve.js`; `web/scripts/build-site.js` + `dev-server.js` + `build-demo-dataset.js` + `patch-demo-mode.js` + `serve-demo.js`. `build-site.js` and `dev-server.js` were also refactored from cwd-dependent to anchored (`import.meta.url` → absolute paths). Workspace `package.json` scripts no longer use `cd ..`. Root `bff:dev` and `demo:build` rewired to delegate via `--workspace=…` instead of invoking workspace `.js` files directly. Dockerfile, `e2e/playwright.config.ts`, `src/test/dockerfile-demo-assets.test.ts`, and various doc cross-references updated. `scripts/` root holds two files now (`check-changeset.js`, `open-sqlite-browser.js`) — both genuinely cross-workspace.

Empty changeset: `@skill-map/spec` is touched only in `index.json` (regenerated to catch up the `conformance/coverage.md` hash bumped by the doc reference fix) and `package.json#scripts` (not in `files:`). `@skill-map/cli` is touched only in `package.json#scripts` and the relocated build/dev tooling. No published surface changes — schemas, prose contracts, conformance suite, CLI binary, verbs, kernel exports are unchanged. `@skill-map/testkit` is untouched. Per the `.changeset/README.md` "non-semver-relevant" guidance, this commit ships an empty changeset to satisfy the `check-changeset.js` CI gate without bumping any version.
