Features

Six checks that turn agent-soup into a map you can read.

Built for repos with hundreds of skills, dozens of agents, and the kind of cross-file coupling that makes grep stop being enough.

Recursive scan

scan

Walks any folder of Markdown, JSON, or YAML and indexes every skill, agent, hook, command, and note in one pass.

Cross-references

graph

Detects every link between files — invocation phrases, file imports, command calls, fetched URLs.

Trigger collisions

collisions

Two skills claiming the same invocation? skill-map flags it before your agent picks the wrong one at runtime.

Orphan detection

orphans

Surfaces files that are defined but never referenced. Dead code, but for your agent ecosystem.

Token weight

weight

Per-node byte and token counts. Find the bloated skill that's eating your context window.

External deps

deps

Lists every URL and fetched asset your skills reach for. Audit your supply chain.

Ecosystem

Six hooks.
Infinite extensions.

The kernel understands six plugin types. Compose what you need, publish your own, install everyone else's. The graph stays the same; the ecosystem makes it yours.

plugin type BUNDLED

Adapter

Recognize a platform

Decides which files in a project are nodes and what kind (skill / agent / hook / command / note). The Claude adapter ships built-in; write one for Cursor, Cline, your own runner.

export const claude: IAdapter = {
  id: 'claude', kind: 'adapter',
  globs: ['~/.claude/**/*.md'],
  classify: (p) => /* skill | agent | hook | … */,
};
plugin type BUNDLED

Detector

Extract signals from a node

Reads the body of a single node and returns Link[] — slash invocations, @-mentions, imports, URLs, anything you can grep. Detectors are how the graph stops being a directory and starts being a graph.

export const slash: IDetector = {
  id: 'slash', kind: 'detector',
  detect: (ctx) => [...ctx.body.matchAll(SLASH_RE)]
    .map(m => ({ kind: 'invokes', target: m[1] })),
};
plugin type BUNDLED

Rule

Lint the whole graph

Receives the assembled graph and returns Issue[]. Trigger collisions, orphans, external deps that died, dangling references — write one rule, run it on every project, ship it as a package.

export const triggerCollisions: IRule = {
  id: 'trigger-collisions', kind: 'rule',
  evaluate: (graph) => byTrigger(graph)
    .filter(([, ns]) => ns.length > 1)
    .map(toCollisionIssue),
};
plugin type

Action

Operate on nodes

Two flavors. Local: deterministic code that mutates files. Invocation-template: a prompt the agent runner executes against an LLM. Either way, actions are how Issues become fixes.

export const renameTrigger: IAction = {
  id: 'rename-trigger', kind: 'action', mode: 'local',
  run: async (node, { newTrigger }) => {
    node.frontmatter.trigger = newTrigger;
    await node.save();
  },
};
plugin type BUNDLED

Audit

Compose rules + actions

A deterministic workflow that runs rules, runs actions, and emits a structured report. CI-friendly. Ship one as a package, the team runs `sm audit @org/security` and gets the same answer every time.

export const securityBaseline: IAudit = {
  id: 'security-baseline', kind: 'audit',
  rules: ['external-deps', 'dangling-refs'],
  run: (graph) => composeRules(graph),
};
plugin type BUNDLED

Renderer

Serialize the graph

Take the in-memory graph and emit it as something else: ASCII, Mermaid, DOT, JSON, your custom format. Bundled renderers cover the basics; the interesting ones are the ones you write.

export const mermaid: IRenderer = {
  id: 'mermaid', kind: 'renderer',
  render: (graph) => 'graph LR\n' +
    graph.edges.map(e => `  ${e.src} --> ${e.dst}`).join('\n'),
};

How it works

Three steps. No config file.

  1. 01

    Point it at a folder.

    One command. ~/.claude/skills, your repo root, any folder — anything with skills, agents, hooks, or notes inside.

    $ sm scan ~/.claude/skills
  2. 02

    Get the graph.

    Static analysis builds the full reference graph. Trigger collisions, orphans, byte weights, external deps — surfaced in milliseconds.

    $ 1,247 nodes · 3,891 edges · 287ms
  3. 03

    Open the explorer.

    Visual graph in your browser. Click any node to inspect. Filter by type. ⌘K to jump anywhere. Or stay in the terminal — your call.

    $ sm open → http://localhost:7800
