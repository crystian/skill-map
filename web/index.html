Features

Six checks that turn agent-soup into a map you can read.

Built for repos with hundreds of skills, dozens of agents, and the kind of cross-file coupling that makes grep stop being enough.

Recursive scan

scan

Walks any folder of Markdown, JSON, or YAML and indexes every skill, agent, hook, command, and note in one pass.

Cross-references

graph

Detects every link between files — invocation phrases, file imports, command calls, fetched URLs.

Trigger collisions

collisions

Two skills claiming the same invocation? skill-map flags it before your agent picks the wrong one at runtime.

Orphan detection

orphans

Surfaces files that are defined but never referenced. Dead code, but for your agent ecosystem.

Token weight

weight

Per-node byte and token counts. Find the bloated skill that's eating your context window.

External deps

deps

Lists every URL and fetched asset your skills reach for. Audit your supply chain.

How it works

Three steps. No config file.

  1. 01

    Point it at a folder.

    One command. ~/.claude/skills, your repo root, any folder — anything with skills, agents, hooks, or notes inside.

    $ sm scan ~/.claude/skills
  2. 02

    Get the graph.

    Static analysis builds the full reference graph. Trigger collisions, orphans, byte weights, external deps — surfaced in milliseconds.

    $ 1,247 nodes · 3,891 edges · 287ms
  3. 03

    Open the explorer.

    Visual graph in your browser. Click any node to inspect. Filter by type. ⌘K to jump anywhere. Or stay in the terminal — your call.

    $ sm open → http://localhost:7800

CLI first

The terminal is the surface. The browser is optional.

Built for people who never closed iTerm. Pipe scan output to jq, gate CI on collisions, embed it in pre-commit hooks.

--json --watch --filter type=skill --no-color --max-depth=3 --ignore-orphans
