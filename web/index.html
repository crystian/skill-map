§ 01

the problem nobody owns

Devs working with AI agents accumulate dozens of skills, agents, commands, and loose docs. Nobody sees the whole picture.

[01]

what lives where

no single place lists every skill, agent, command, hook, or note across the ecosystem.

[02]

who calls whom

cross-file references, invocations, and dependencies are invisible until they break.

[03]

trigger collisions

two skills with overlapping triggers silently fight. you only notice when the wrong one wins.

[04]

orphans & dead weight

files nothing references. drafts that never shipped. token cost you keep paying.

[05]

external dependencies

what skills pull from which sub-agents, mcps, or external services — and where they leak.

[06]

last optimized when

no history of when each skill was validated, optimized, or last touched. until now.

§ 02

three views, one graph

List, graph, and inspector — every view reads the same kernel data and stays in sync.

drop web/img/list.png

searchable list with filters by kind, stability, and issues. fastest way to find anything.

drop web/img/graph.png

force-directed graph rendered with foblex flow. pan, zoom, color by kind, click to inspect.

drop web/img/inspector.png

read-only frontmatter, body preview, references, and incoming/outgoing links — for the selected node.

§ 03

how it works

Four pieces. The kernel knows nothing about platforms — everything else plugs in.

      files            kernel           cli · sm           ui
        │                │                  │                  │
        │  scan          │  ports/adapters  │  json out        │
        ├───────►───────┤◄────────────────┤                  │
        │                │                  │                  │
        │                │  6 extension     │                  │
        │                │  kinds           ├────────►────────┤
        │                │                  │                  │  render
        │                │                  │                  │
   .md/.markdown      hexagonal core     headless cli       angular spa

scanner emits structured nodes/links/issues → kernel applies rules → cli serves json → ui renders.

§ 04

quickstart

One install. One scan. Zero LLM calls unless you ask for them.

$ npm i -g @skill-map/cli
$ sm scan
$ sm inspect path/to/skill.md
$ sm issues --json
  • pure typescript · node ≥ 20 · offline by default.
  • spec is published on npm — build alternative implementations.
  • plugins drop into .skill-map/plugins/. six extension kinds, all stable.
§ 05
